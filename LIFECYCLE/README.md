# Lifecycle

The chronological path a game moves through, kickoff to postmortem. Read `FOUNDATION/` first
(how we think — it applies to every phase); then these phases in order. The OS literally walks
you through development.

1. **[Project](1-PROJECT/)** — define the game
2. **[Design](2-DESIGN/)** — design the experience
3. **[Engineering](3-ENGINEERING/)** — set the standards
4. **[Implementation](4-IMPLEMENTATION/)** — build it
5. **[Playtesting](5-PLAYTESTING/)** — test with real players
6. **[Production](6-PRODUCTION/)** — ship it
7. **[Postmortems](7-POSTMORTEMS/)** — learn, and feed it back into FOUNDATION

Cross-cutting layers live *outside* these numbered phases because they apply throughout:
`AI/` (collaboration), `FOUNDATION/Decision-Framework.md` (the log), and
`FOUNDATION/Feature-Lifecycle.md` (the per-feature pipeline that spans phases).

A phase folder with only a README is one we haven't written down yet — not an empty stub. It
grows as real games generate content.
