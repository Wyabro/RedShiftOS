# 7 · Postmortems

**Learn from every shipped project.** This is the feedback loop that makes the OS smarter —
the reason "the system gets smarter because we actually shipped with it."

- After a project (or a meaningful milestone), write a postmortem: what went well, the
  mistakes, the AI failures, the architecture regrets, and the things worth repeating.
- Turn each durable lesson into a new `L-NN` entry in `FOUNDATION/Lessons-Learned.md`, and each
  recurring smell into a `FOUNDATION/Anti-Patterns.md` entry.
- One folder per project (e.g. `cart-clash/`) as they're written.

The head-only lessons — the design fights and playtest gut-punches that never hit git —
accrete here as you build, not up front.
