# 2 · Design

**Design the experience before implementing it.** Player value, core loop, pillars, and the
*feeling* — decided before anyone writes code.

- Say the feeling before you ask for code (`FOUNDATION/Feature-Lifecycle.md` → "Say the
  feeling before you build").
- Every feature names the pillar it serves, or it doesn't get built (Manifesto §6;
  `FOUNDATION/Anti-Patterns.md` → Feature Creep).

Grows with: design pillars, core-loop notes, and UX principles as games define them. Until
then, the binding guidance is the Manifesto + the Feature Lifecycle's design stage.
